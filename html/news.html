March 2002: Plans for Gambit 0.97

Version 0.97 of Gambit has been under (irregular) development for almost two years, and is nearing release. Our current target release date is sometime in Summer 2002. Some features of the new version:
  • Algorithms: These algorithm enhancements will provide new, more useful tools for computing Nash equilibria in games with three or more players.
    • A new path-following algorithm for computing quantal response equilibria, based on a paper which can be downloaded from Ted's homepage. This algorithm is capable of tracing the quantal response correspondence for much greater values of lambda than before, making it possible to identify the limiting point of the principal branch of the correspondence for most games.
    • Implementation of Yamamoto's algorithm (IJGT, 1993) for finding a proper equilibrium in a normal form game.
  • Interface: The graphical interface has been completely reimplemented, moving from wxWindows version 1.68 to the current version of wxWindows 2.xx. The interface has been reorganized to be more standardized. Versions are under development for both Microsoft Windows and Linux (using either the GTK+ or the Motif/Lesstif libraries).
  • Command language: The command language is largely unchanged from the previous version. A new graphical window interface to the command language will be available on all platforms, replacing the MFC-based Microsoft Windows interface in 0.96. Additionally, a command-line interface is available under Windows now for better batch-file support.
  • PXI: The PXI program for plotting and analyzing quantal response equilibria has also been reimplemented with wxWindows 2.xx, with a more modern and clean interface. Improved features for experimental data entry and manipulation are included.
  • Additionally, many minor bugs have been squashed, and new features added based upon feedback from users.
Watch this space for more details on the release date.

November 28, 2000: Gambit 0.96.3 released

Bug fixes:
  • Null outcomes in normal form files are now correctly read.
  • Extensive form window sizes correctly in games with many multi-node information sets. (In previous versions, outcome payoffs could not always be read in such games.)
  • Chance probabilities are copied after infoset-cut.
  • The return type for the GCL function MoveToInfoset[] is corrected.
  • Emptied information sets are now automatically deleted in the GUI when editing the extensive form.
  • The Unix Makefile has been modified to use $(MAKE) for its recursive calls.
  • All parsers have been changed to work with bison, instead of bison++.
  • Linux binary now linked statically. So motif need not be installed locally to run the GUI.

February 2000: Gambit 0.96 released

Feature list:
  • New polynomial code for computation of all Nash equilibria of n-person games
  • Added clique code (thanks to Bernhard von Stengel) for characterizing the complete set of Nash equilibria in non generic two person games
  • Command line editor added to GCL
  • Improved tableau and linear programming code for speeding up Rational precision computations
  • New Number Class added to support mixed Floating point and Rational computaations
  • Added exception handling to stabilize code
  • Fixed many bugs from release .94
