November 28, 2000: Gambit 0.96.3 released

Bug fixes:
  • Null outcomes in normal form files are now correctly read.
  • Extensive form window sizes correctly in games with many multi-node information sets. (In previous versions, outcome payoffs could not always be read in such games.)
  • Chance probabilities are copied after infoset-cut.
  • The return type for the GCL function MoveToInfoset[] is corrected.
  • Emptied information sets are now automatically deleted in the GUI when editing the extensive form.
  • The Unix Makefile has been modified to use $(MAKE) for its recursive calls.
  • All parsers have been changed to work with bison, instead of bison++.

March 2000: Gambit 0.96.2 released

Bug fixes:
  • Bug in File select dialog fixed
  • Fixed bug in installation script

February 2000: Gambit 0.96 released

Feature list:
  • New polynomial code for computation of all Nash equilibria of n-person games
  • Added clique code (thanks to Bernhard von Stengel) for characterizing the complete set of Nash equilibria in non generic two person games
  • Command line editor added to GCL
  • Improved tableau and linear programming code for speeding up Rational precision computations
  • New Number Class added to support mixed Floating point and Rational computaations
  • Added exception handling to stabilize code
  • Fixed many bugs from release .94
